## プロダクト#1: Elio - Product Huntに無視されたオフラインAI

![画像: 飛行機内でElioを実行しているノートPCのスクリーンショット]

**問題:**

私は頻繁に旅行します。長時間のフライト、ホテルの不安定なWiFi、インターネットが高額または存在しない遠隔地。しかし、コーディング、ライティング、リサーチなど、すべてにおいてAIアシスタントに依存するようになっていました。

ある日、東京への14時間のフライトで立ち往生しました。インターネットなし。ChatGPTなし。Claudeなし。私とノートPCだけで、14時間の潜在的な生産性が無駄になりました。

私は思いました:**「なぜAIはデバイス上で動作できないのか?」**

**解決策:**

Elioは**デバイス上で動作するLLM**で、完全にローカルマシン上で実行されます。インターネット不要。データはデバイスを離れません。プライベートで、高速で、どこでも動作します。

**技術的課題:**

Elioの構築は、今まで遭遇したことのない問題を解決することを意味しました:

1. **モデルサイズ**: パフォーマンスを破壊せずにRAMに収まる有能なLLMを取得すること
2. **推論速度**: コンシューマーハードウェアで使用可能な速度で応答を生成すること
3. **量子化**: 品質を維持しながらモデルサイズを削減すること
4. **クロスプラットフォーム**: MacOS(ARM + Intel)、Windows、Linuxで動作すること

**Rust**で構築した理由:
- メモリ安全性(メモリリークによるクラッシュがない)
- パフォーマンス(より良い人間工学でC言語に近い速度)
- 小さなバイナリサイズ(アプリ全体が50MB未満)

**Product Huntの惨事:**

ローンチ日が来ました。緊張していましたが興奮していました。すべてを「正しく」やりました:
- ✅ 説得力のあるタグライン
- ✅ デモビデオ
- ✅ 明確な価値提案
- ✅ すべてのコメントに返信

結果:**6票のupvote**。

技術的洗練度が低く、独創性が低く、有用性が低い製品が何百ものupvoteを獲得するのを見ました。私は打ちのめされました。

**学んだこと:**

1. **Product Huntは検証指標ではない** - 抽選と人気コンテストの混合物です
2. **マーケティングはコードと同じくらい重要** - 時間の99%を構築に、1%を人に伝えることに費やしました
3. **タイミングがすべて** - エンゲージメントが低い忙しい週にローンチしました
4. **コミュニティファースト** - 既存のオーディエンス、メールリスト、Twitterフォロワーがありませんでした
5. **プロダクトは死んでいない** - 6人が気にかけていたので、続ける理由は十分でした

ElioはProduct Huntで勝利しませんでした。しかし、**外部の検証を必要とせずにプロダクトをローンチする方法**を教えてくれました。

## プロダクト#2: ChatWeb.ai - 音声ファーストのブラウザ自動化ツール

![画像: 音声波形アニメーション付きChatWeb.aiインターフェースのスクリーンショット]

**問題:**

毎日、同じ反復作業を行っていました:
- AI最新情報のために複数のニュースサイトをチェック
- ツール間で情報をコピー&ペースト
- 異なるソース間で類似情報を検索
- 類似データでフォームに入力

手がタイピングで痛くなりました。目が読書で疲れました。そして常に思っていました:**「これは自動化されるべきだ。」**

しかし既存のツールは:
- 複雑すぎる(Seleniumスクリプト)
- 制限が多い(IFTTT/Zapierは複雑なフローを処理できない)
- 高額すぎる(エンタープライズRPAツール)

**解決策:**

ChatWeb.aiは、**音声でブラウザを制御**し、**AIが複雑なワークフローを実行**できるようにします。

10のウェブサイトでトピックを調査したいですか? 頼むだけです。
競合他社のサイトからデータを抽出したいですか? 説明するだけです。
フォーム入力を自動化したいですか? 必要なことを言うだけです。

**技術スタック:**

**サーバーレス**にオールインしました:

```
フロントエンド: Next.js 16 + React 19(高速反復)
バックエンド: AWS LambdaのRust(コールドスタート<100ms)
データベース: DynamoDB(リクエスト毎の課金)
音声: Web Speech API(ブラウザネイティブSTT)
AI: Claude Sonnet 4.5(最も有能なモデル)
ホスティング: Cloudflare Pages(無料層、無制限帯域幅)
```

**なぜサーバーレスにRust?**

みんなが言いました:「LambdaにはNode.jsかPythonを使え。」

私がRustを選んだ理由:
1. **コールドスタート**: 約90ms対Node.jsの約500ms
2. **メモリ**: 128MB対Pythonの512MB以上
3. **コスト**: 100万リクエストあたり¥20対Pythonの¥500以上
4. **パフォーマンス**: 単一インスタンスで1000+ req/sを処理

12ヶ月間で、この選択により**AWSコストで約¥2,400節約**できました。

**音声機能:**

**音声入力**を追加した理由:
- コーディング中に使用(手はキーボード上)
- 歩行/運動中に動作
- 複雑なリクエストに自然
- アクセシビリティの勝利

**Web Speech API**(無料、ブラウザネイティブ、サーバーコストなし)を使用しています。

**成長ストーリー:**

- **1ヶ月目**: 12ユーザー(友人がテスト)
- **3ヶ月目**: 47ユーザー(Redditの投稿が注目を集める)
- **6ヶ月目**: 134ユーザー(口コミ)
- **12ヶ月目**: 289ユーザー(月次15%成長)

**現在のMRR**: ¥87,000(~$650 USD)
- スタータープラン: ¥900/月($9 USD) - 189ユーザー
- プロプラン: ¥2,900/月($29 USD) - 23ユーザー

**重要な学習:**

人々は機能を買いません。**節約された時間**を買います。

ChatWebを「AIを使ったブラウザ自動化」として売り込んだとき、誰も気にしませんでした。

## プロダクト#3: News.xyz - 情報過多のためのAIニュースアグリゲーター

![画像: キュレーションされたAIニュースフィードを表示するNews.xyzダッシュボード]

**問題:**

私は47のニュースレターを購読していました。はい、**47個**です。

毎朝:47通のメール。毎晩:さらに47通の更新。

- TechCrunch
- Hacker Newsダイジェスト
- AI Breakfast
- ImportAI
- Benedict Evans
- Stratechery
- TLDR
- ...および40以上

情報に溺れていましたが、**洞察**に飢えていました。

**解決策:**

News.xyzは**AIを活用したニュースアグリゲーター**で、以下のことを行います:
1. すべてのニュースレターを読む
2. 重複を削除(同じストーリー、12の異なるソース)
3. 主要なポイントを要約
4. 本当に重要なことだけを含む**1通のメール**を送信

**技術アーキテクチャ:**

```
メール解析: Postal MIME(Rustライブラリ)
重複排除: 埋め込み類似性(OpenAI)
要約: Claude Haiku(高速+安価)
配信: Resend API(トランザクショナルメール)
ストレージ: PostgreSQL(Supabase)
```

**重複排除の課題:**

問題:同じストーリーがニュースレター間で15回出現します。

解決策:**ベクトル埋め込み**
1. 各記事の埋め込みを生成(1536次元)
2. コサイン類似度を比較
3. 0.92以上の類似度を持つ記事=重複
4. 最も包括的なバージョンを保持

これにより**ノイズを73%削減**しました。

**ユーザー成長:**

- **ローンチ**: 8ユーザー(個人的ネットワーク)
- **3ヶ月目**: 89ユーザー(ShowHN投稿)
- **6ヶ月目**: 267ユーザー(Product Hunt - はい、再挑戦しました)
- **12ヶ月目**: 506ユーザー(オーガニック+紹介)

**収益化:**

- 無料層:10のニュースレター、毎日のダイジェスト
- プロ層(¥500/月):無制限のニュースレター、リアルタイム更新、カスタムフィルター

**現在のMRR**: ¥43,000(~$320 USD)、86名のプロユーザーから

**重要な洞察:**

人々は**痛みを減らす**ために支払い、機能を追加するためではありません。プロプランは「10ニュースレター制限」を削除しましたが、これが最大の痛点でした。

¥500/月(週にコーヒー1杯未満)という価格設定により、考える必要がありませんでした。

## 技術スタックの深掘り:これらの選択が重要な理由

![画像: 4つのプロダクトすべての技術スタックを示すアーキテクチャ図]

人々は尋ねます:*「なぜこんなに多くの異なる技術を?」*

答え:**各プロダクトには異なる制約があります。**

### なぜRust?

私はRustを以下に使用します:
- **ChatWeb.aiバックエンド**:パフォーマンス+低メモリ=安価なLambda
- **News.xyz解析**:ユーザーメールを処理する際のメモリ安全性
- **JiuFlow分析**:500ms未満で100k以上のレコードを処理

**利点:**
- ⚡ **速度**:Python/Node.jsの10-100倍高速
- 💰 **コスト**:AWSの請求額が80%安い
- 🔒 **安全性**:本番環境でランタイムクラッシュなし
- 📦 **サイズ**:小さなバイナリ=高速デプロイ

**トレードオフ:**
- ⏰ 開発時間が長い(厳格なコンパイラ)
- 📚 学習曲線が急
- 🔧 JS/Pythonエコシステムよりライブラリが少ない

**ROI**: 12ヶ月間で、Rustにより**インフラコストで約¥3,600節約**できました。

### なぜNext.js?

すべてのフロントエンドにNext.jsを使用する理由:
- **React 19**:よく知っているコンポーネントモデル
- **App Router**:高速ページロード、自動コード分割
- **Server Components**:バンドルサイズの削減
- **Edge Runtime**:どこでもデプロイ(Cloudflare、Vercel、Fly.io)

**反復速度**は完璧なアーキテクチャよりも重要です。Next.jsは機能を**高速に**出荷できます。

### なぜSupabase?

Supabase = PostgreSQL + Auth + Storage + Realtime

以下を構築する代わりに:
- ユーザー認証
- データベース管理
- ファイルストレージ
- リアルタイムサブスクリプション

それらすべてを1つのサービスで取得します。**バックエンド作業の100時間以上を節約**します。

**コスト**: 4つのプロダクトすべてで月¥2,500。AWS RDS + Cognito + S3では月¥20,000以上かかります。

### インフラストラクチャの選択

| プロダクト | ホスティング | 理由 |
|---------|---------|-----|
| Elio | ネイティブアプリ | デバイス上で動作、サーバー不要 |
| ChatWeb.ai | AWS Lambda + Cloudflare | サーバーレススケール、エッジキャッシング |
| News.xyz | Fly.io | ユーザーに近い(アジア)、永続的ワーカー |
| JiuFlow | Fly.io | リアルタイム分析、低レイテンシ |

**コスト最適化:**

総インフラストラクチャコスト:**月¥8,400**(~$63 USD)

内訳:
- Cloudflare Pages:¥0(無料層)
- AWS Lambda:¥2,100(リクエスト毎の支払い)
- Fly.io:¥3,800(2つのアプリ、$28 USD)
- Supabase:¥2,500(1つのデータベース、共有)

**利益率**: 97% 💰

## DAO哲学:なぜコミュニティとして構築したか

![画像: EBRトークンビジュアライゼーション付きEnablerDAOロゴ]

**なぜDAO?**

ほとんどの起業家は株式を非公開にします。私はEnablerDAOのために**分散型自律組織**構造を選択しました。

理由は以下の通りです:

### 1. デフォルトで透明性

すべての主要な決定は公開されています:
- 収益数値(毎月公開)
- ロードマップの優先順位(コミュニティ投票)
- 資金残高(オンチェーン、検証可能)

これによりユーザーとの**信頼**が構築されます。彼らは正確に何をサポートしているかを知っています。

### 2. EBRトークンによるコミュニティ所有権

**EBR(Enabler)トークン** = ガバナンス+報酬

EBRの獲得方法:
- **製品を使用**:デイリーアクティブユーザーは1日10 EBR獲得
- **貢献**:バグレポート、機能アイデア、ドキュメント
- **拡散**:コンバートする紹介
- **構築**:オープンソース貢献

EBRでアンロックされるもの:
- **ガバナンス**:製品ロードマップに投票
- **報酬**:収益分配(利益の10%)
- **アクセス**:ベータ機能、優先サポート
- **割引**:プロプランが50%オフ

### 3. 整合したインセンティブ

従来のモデル:会社が利益を最大化→ユーザーがより多く支払う

DAOモデル:コミュニティがエコシステムを成長させる→全員が利益を得る

**例:**

JiuFlowが1,000ユーザーに達したとき、早期採用者に100,000 EBRを配布しました。これらのトークンは現在、JiuFlowのロードマップに投票権を持っています。

結果:**ユーザーが実際に望む機能が構築される**、私が望むと思うものではなく。

### 4. 持続可能性

DAOは**長期的思考**のために設計されています。VCの10倍になるか死ぬかのプレッシャーはありません。買収交渉もありません。ただ持続可能な成長だけです。

**現在の資金**:5,000,000 EBR(将来の貢献者のために予約)

## 学んだ教訓:12ヶ月と4つのプロダクトが私に教えてくれたこと

![画像: 手書きのノートとコードスニペット付きジャーナル]

### 1. Product Huntは重要ではない(しかしローンチは重要)

**私が思ったこと**:Product Huntの成功=製品検証

**現実**:Product Huntの成功=マーケティング+タイミング+運

**重要なこと**:製品を**必要とする1人の前**に出し、彼らのフィードバックを聞くこと。

ElioはProduct Huntで6票でしたが、**2,400以上のダウンロード**があります。その6票は成功を予測しませんでした。

### 2. コミュニティファースト、機能セカンド

私は6ヶ月間Elioをプライベートで構築しました。コミュニティゼロ。ユーザーゼロ。フィードバックゼロ。

JiuFlowでは、BJJ Discordサーバーで**初日のスクリーンショット**を共有しました。ローンチする前に50ユーザーを獲得しました。

**学習**:公開で構築する。早く出荷する。準備ができたと思う**前に**ユーザーを巻き込む。

### 3. 完璧は出荷の敵

私の最大の後悔? ローンチするのに長く待ちすぎたこと。

- Elio:「完璧なオフラインモデル」を待つ(3ヶ月遅延)
- ChatWeb:「すべての音声コマンド」を待つ(2ヶ月遅延)
- News.xyz:バグ付きでローンチ、ユーザーレポートに基づいて修正(最良の決定)

**学習**:80%で出荷する。最後の20%はしばしば重要ではない。

### 4. コストではなく価値に基づいて価格設定

最初、「コストをカバー+小さなマージン」なので、ChatWebを¥300/月で価格設定しました。

ユーザーは私のコストを気にしませんでした。彼らは**1日2時間節約**を気にしました。

2時間×¥3,000/時間(平均知識労働者)=¥6,000/日の価値。

価格を¥2,900/月に上げました。価値が明白だったので、**コンバージョンが上がりました**。

**学習**:価格は価値を伝える。安すぎる=「これはおそらく機能しない。」

### 5. 速く反復、速く失敗

すべてのプロダクトは**間違った仮定**から始まりました:

- **Elio**:「人々はプライバシーを望む」→実際には**オフライン機能**を望んでいた
- **ChatWeb**:「人々は自動化を望む」→実際には**時間節約**を望んでいた
- **News.xyz**:「人々はより多くの情報を望む」→実際には**ノイズを減らす**ことを望んでいた
- **JiuFlow**:「人々はテクニックライブラリを望む」→実際には**進捗の可視性**を望んでいた

私は**出荷して聞く**ことによってのみ真実を発見しました。

**学習**:最初のアイデアはおそらく間違っています。とにかく出荷して、ユーザーに実際に必要なものを伝えてもらいます。

### 6. オープンソースは信頼を生む

ElioはGitHubで**完全にオープンソース**です。収益ゼロ。

しかし、それは:
- AIコミュニティで私の評判を構築
- 有料製品へのトラフィックを促進
- 信頼を生成(「彼はコードを共有する、彼は正当であるに違いない」)
- それを改善した貢献者を引き付けた

**学習**:専門知識を提供する。それは倍増して戻ってきます。

### 7. 多様化=回復力

4つのプロダクトを持つことは:
- 1つが遅い月を持つ場合、他が補償
- 異なるオーディエンス=より広いリーチ
- 共有インフラストラクチャ=規模の経済

**しかし**また意味する:
- 4倍のメンテナンス負担
- プロダクト間で注意が分散
- どれも深く掘り下げることができない

**学習**:2-3製品が最適です。4は限界です。

### 8. データは嘘をつかない

JiuFlowは私がハーフガードを避けていることを示しました。データと議論できませんでした。

製品も同じ:
- **定着データ**はChatWebの音声機能が3倍の定着率を持つことを示した
- **使用データ**はNews.xyzユーザーがモバイルアプリを望んでいることを示した(現在構築中)
- **収益データ**はジムライセンスが個人サブスクリプションより10倍良いことを示した

**学習**:すべてを計測する。毎週データを見る。それが決定を導くようにする。

### 9. ソロ起業家の持続可能性

私はこれらすべてを**ソロで**構築しました。共同創設者なし。チームなし(まだ)。

**可能にしたこと:**
- Rust(バグが少ない=メンテナンスが少ない)
- サーバーレス(管理するサーバーなし)
- Supabase(構築するバックエンドなし)
- Next.js(高速反復)

**難しかったこと:**
- 決定について議論する相手がいない
- すべてのカスタマーサポート=私
- すべてのコード=私
- すべてのマーケティング=私

**学習**:ソロは可能ですが、天井があります。¥500K MRRで、助けを雇います。

### 10. BJJマインドセットはスタートアップに適用される

- **早くタップ、頻繁にタップ**:失敗したプロダクトはデータ、敗北ではない
- **一貫して現れる**:毎日の進歩が散発的な英雄的行為に勝る
- **不快感を受け入れる**:最高の成長は快適ゾーンの外で起こる
- **力よりテクニック**:スマートなコードがブルートフォースに勝る
- **より良い人とローリング**:より良いビルダーから学ぶ

**学習**:マットが会社を構築する方法を教えてくれました。
