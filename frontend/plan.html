// nanobot とは

chatweb.ai のコアエンジン。複数のLLM（GPT-4o, Claude, Geminiなど）を 同時比較できるマルチモデルプラットフォームをRustで実装したもの。 高速・軽量・低コストが特徴で、AI開発者・研究者が主なユーザー。 enabler-cli がこのリポジトリを自動改善し、PRを作成します。

// 改善タスク一覧（AI自動実行）

#1 レスポンスキャッシュの実装
同一プロンプトへの重複APIコールを削減。Redis または in-memory LRU キャッシュで応答速度を改善し、APIコストを最大40%削減。
performance backend Rust
100-150 EBR
#2 エラーハンドリングの強化
各LLM APIのタイムアウト・レート制限エラーを適切に処理。ユーザーへの分かりやすいエラーメッセージと自動リトライ機構を追加。
reliability ux
80-120 EBR
#3 ストリーミングレスポンスの最適化
SSE（Server-Sent Events）のバッファリング処理を改善。大きなレスポンスでもUIがフリーズしないよう非同期処理を最適化。
streaming async
70-100 EBR

現在のカバレッジ約35%を70%以上に引き上げ。特にAPI統合テストと境界値テストを追加。CI/CDパイプラインに組み込む。
